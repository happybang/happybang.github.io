通过requireTemplate提高模板引擎的性能

#关于模板引擎：
1.模板引擎的优点：编写代码直观，代码方便管理,数据和展示逻辑解耦等。
2.模板引擎的缺点：性能上多了一次解析过程，对性能控不能忍受。

#解决思路：

通过构建来解决该问题： 将第一次模板引擎解析放在构建的时候来做。

##拿ejs模板引擎举例：

###下面列举其在运行时执行步骤：
1.首先我们写了一个模板内容如下：

<% if (names.length) { %>


    <% names.forEach(function(name){ %>

<li foo=’<%= name + “‘“ %>’><%= name %>
<% }) %>

<% } %>

2.我们的代码如下：

var users = [];
users.push({ name: ‘Tobi’, age: 2, species: ‘ferret’ })
users.push({ name: ‘Loki’, age: 2, species: ‘ferret’ })
users.push({ name: ‘Jane’, age: 6, species: ‘ferret’ })
var ret = ejs.render(str, {//此str为上一步的模板字符串
users: users
});
console.log(ret);
3.最后console输入的结果为:

<ul>

  • Tobi

    • <li foo='Loki'>Loki</li>
  • Jane

    • `

    ###那么ejs为我们做了什么事情呢？
    ejs通过解析上面的模板生成如下的代码：
    （伪代码）

    var _content = [];
    var __append =function(a){
    _content .push(a);
    }
    with（data){
    _append(“ul”)
    names.forEach(function(name){
    _append(“

  • “+name+”
    • “)
    }
    }
    return _content.join(‘’);

    并将users数据带入上面代码的context中执行得到最终结果。。

    3、ok知道了ejs在runtime如何工作，设想一个场景如下，针对上述例子进行改进：
    首先模板书写方式不变。
    直接修改第二部分的脚本：

    var users = [];
    users.push({ name: ‘Tobi’, age: 2, species: ‘ferret’ })
    users.push({ name: ‘Loki’, age: 2, species: ‘ferret’ })
    users.push({ name: ‘Jane’, age: 6, species: ‘ferret’ })
    var ejsrun=@requireTemplate(str);//此处的str仍然是模板字符串或者路径
    var ret = ejsrun({
    users:users
    })
    console.log(ret);

    然后执行构建工具，
    之后deploy的代码如下：

    })
    console.log(ret);
    以上抛砖引玉拿ejs模板举例。
